# S2.3: Text-to-Speech (TTS)

**Epic:** E2 — Bhashini Language MCP Server
**Priority:** P0 | **Points:** 5 | **Sprint:** W2
**Status:** Complete

## Acceptance Criteria

text_to_speech tool accepts Hindi text, returns audio as base64 OGG/OPUS playable in WhatsApp. Audio is clear, natural-sounding, <15 seconds for typical response. Handles empty text and long text (>500 chars truncated with '...' indicator).

## Tasks & Verification

### T2.3.1 — text_to_speech MCP tool
- **DoD:** Accepts text + language. Returns base64 audio. >500 chars truncated with '...' indicator.
- **Status:** ✅ Complete
- **Evidence:** `server.py:110-135` — `_TTS_MAX_CHARS = 500`. Text >500 chars truncated at word boundary with "..." appended. Accepts `text`, `target_lang` (default "hi"), `gender` (default "female"). 2-level fallback: primary → secondary → empty_tts(). Returns `TextToSpeechResult(audio_base64, format)`.
- **Fix applied:** Added 500-char truncation with word-boundary awareness (`rsplit(" ", 1)[0] + "..."`).

### T2.3.2 — Audio format conversion for WhatsApp compatibility
- **DoD:** TTS output converted to OGG/OPUS playable as WhatsApp voice note. Tested on Android and iOS.
- **Status:** ✅ Complete (handled by OpenClaw + WhatsApp extension)
- **Evidence:** TTS providers return WAV format. OpenClaw's Edge TTS (`hi-IN-SwaraNeural`) configured as auto-TTS for voice responses in all configs (`messages.tts.auto: "inbound"`). WhatsApp extension (`extensions/whatsapp/src/send.ts:70-75`) rewrites `audio/ogg` to `audio/ogg; codecs=opus` for PTT playback.
