# S3.1: Mandi Price Lookup

**Epic:** E3 — Government Data MCP Server
**Priority:** P0 | **Points:** 5 | **Sprint:** W2
**Status:** Complete

## Acceptance Criteria

get_mandi_prices tool returns today's prices for a given commodity + state + district from data.gov.in. Returns structured JSON with mandi name, price, date. Handles: commodity not found, district not in dataset, API down. Caches with 6hr TTL.

## Tasks & Verification

### T3.1.1 — Scaffold govdata-india MCP server (FastMCP)
- **DoD:** Empty MCP server starts, connects to OpenClaw as registered skill. Responds to tools/list.
- **Status:** ✅ Complete
- **Evidence:** `server.py` — `FastMCP("govdata-india")` with 4 `@mcp.tool()` functions: `mandi_prices`, `pm_kisan_status`, `weather_info`, `scheme_search`. Runs via `python -m govdata_india.server`.

### T3.1.2 — data.gov.in API client with caching
- **DoD:** HTTP client with API key auth, 10s timeout, retry 1x on 5xx, in-memory cache with configurable TTL.
- **Status:** ✅ Complete
- **Evidence:**
  - `cache.py` — `TTLCache` class with `time.monotonic()` expiry, `get/set/clear` methods.
  - `cache.py` — `fetch_with_retry()` function: 10s timeout, retries once on 5xx, re-raises on non-5xx errors.
  - All three API modules (mandi, pmkisan, weather) use shared `fetch_with_retry`.
- **Fix applied:** Added `fetch_with_retry()` to `cache.py` as shared helper. Previously API calls had no retry logic.

### T3.1.3 — get_mandi_prices tool
- **DoD:** Returns prices for commodity+state+district. Handles misspelled commodity (fuzzy match), district not found (suggest alternatives), no data today (return latest + date). Cache 6hr.
- **Status:** ✅ Complete
- **Evidence:**
  - `mandi.py` — `_fuzzy_match_commodity()` uses exact match → substring match → trigram Jaccard similarity (threshold 0.2). 32 known commodities.
  - Empty results with suggestions: `source` field includes "Try: wheat, barley" when fuzzy match found alternatives.
  - Cache: `_MANDI_TTL = 6 * 3600` (6 hours). Only caches successful results.
  - Response: `MandiPricesResponse(prices=[MandiPrice(mandi, commodity, variety, min_price, max_price, modal_price, date)])`.
- **Fixes applied:** Added fuzzy commodity matching and suggestion alternatives in response.
