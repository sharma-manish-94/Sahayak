# S2.4: End-to-End Voice Pipeline

**Epic:** E2 — Bhashini Language MCP Server
**Priority:** P0 | **Points:** 3 | **Sprint:** W2
**Status:** Complete

## Acceptance Criteria

Full pipeline works: Hindi voice note → ASR → English translation → [agent processing] → Hindi translation → TTS → WhatsApp voice reply. Round-trip <15 seconds for simple queries.

## Tasks & Verification

### T2.4.1 — Wire full voice pipeline
- **DoD:** Hindi voice → Hindi text → English text → placeholder agent → Hindi text → Hindi voice. <15s. 5/5 success.
- **Status:** ✅ Complete
- **Evidence:** All 4 MCP tools wired in `server.py` with provider selection via `LANG_PROVIDER` env var (auto/sarvam/bhashini). Pipeline flow:
  1. `speech_to_text(audio_base64, "hi")` → Hindi text
  2. `detect_language(text)` → language code
  3. `translate(text, "hi", "en")` → English for agent
  4. Agent processes in English via Claude Sonnet
  5. `translate(response, "en", "hi")` → Hindi response
  6. `text_to_speech(hindi_response, "hi")` → audio base64
  7. OpenClaw Edge TTS sends as WhatsApp voice note

### T2.4.2 — Bhashini fallback chain
- **DoD:** ASR fail → Whisper. NMT fail → passthrough. TTS fail → text-only. Each logged.
- **Status:** ✅ Complete
- **Evidence:** `fallback.py` implements three fallbacks:
  - **ASR:** `whisper_transcribe()` (lines 14-44) — uses OpenAI Whisper API via `OPENAI_API_KEY`. Returns empty result with error if key missing.
  - **NMT:** `passthrough_translate()` (lines 47-50) — returns original text unchanged.
  - **TTS:** `empty_tts()` (lines 53-55) — returns empty audio with `format="wav"`. OpenClaw's Edge TTS handles final voice output.
  - Each tool in `server.py` has 3-level chain: primary → secondary provider → fallback.
