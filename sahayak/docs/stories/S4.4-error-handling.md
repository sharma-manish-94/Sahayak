# S4.4: Error & Fallback Handling

**Epic:** E4 — Agent Routing & LLM Integration
**Priority:** P0 | **Points:** 3 | **Sprint:** W3
**Status:** Complete

## Acceptance Criteria

Every failure mode produces a user-friendly message in detected language. Bhashini down → English fallback + apology. data.gov.in down → 'service temporarily unavailable, try again'. LLM timeout → graceful retry once then apologize. No raw error messages ever reach user.

## Tasks & Verification

### T4.4.1 — Error handling for all failure modes
- **DoD:** (1) Bhashini timeout → English fallback (2) data.gov.in 500 → 'service unavailable' (3) Claude timeout → retry+apologize (4) Unknown language → English response. No raw errors/stack traces/JSON to user.
- **Status:** ✅ Complete
- **Evidence:**

  **Structural error handling (MCP servers):**
  - All API calls wrapped in try/except returning structured Pydantic responses, never exceptions.
  - `fetch_with_retry()` in `cache.py`: retries once on 5xx before returning error.
  - `mandi.py`: returns `MandiPricesResponse(prices=[], source="error: ...")`.
  - `pmkisan.py`: returns `PMKisanResponse(message="Error fetching PM-KISAN data: ...")`.
  - `weather.py`: returns `WeatherResponse(message="Error fetching weather data: ...")`.
  - All timeouts: 10s via httpx (`timeout=10.0`).

  **Language service fallbacks (bhashini-lang):**
  - ASR: primary → secondary provider → Whisper (`fallback.py:14-44`).
  - NMT: primary → secondary → passthrough (`fallback.py:47-50`).
  - TTS: primary → secondary → empty audio (`fallback.py:53-55`).

  **User-facing error localization (agent prompt):**
  - `SYSTEM.md` Error Handling section provides Hindi + English templates for every failure mode:
    - API timeout/down → "सरकारी डेटा सेवा अभी उपलब्ध नहीं है..."
    - No data found → "इस जिले/फसल के लिए डेटा उपलब्ध नहीं है।"
    - Voice not recognized → "आवाज़ स्पष्ट नहीं हुई, कृपया टाइप करके भेजें।"
    - Unknown language → English: "I work best in Hindi and English."
    - LLM/tool error → "कुछ तकनीकी समस्या है, कृपया थोड़ी देर बाद कोशिश करें।"
  - Agent instructed: "Never show raw errors, stack traces, or JSON to the user."
  - **Fix applied:** Added Error Handling section to `SYSTEM.md` with bilingual error templates.
