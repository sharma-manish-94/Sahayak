# S1.2: WhatsApp Channel Integration

**Epic:** E1 — OpenClaw Foundation & Security
**Priority:** P0 | **Points:** 3 | **Sprint:** W1
**Status:** Complete

## Acceptance Criteria

Can send and receive text messages via WhatsApp personal number through OpenClaw gateway. Voice notes received as audio files accessible to skills.

## Tasks & Verification

### T1.2.1 — Configure WhatsApp personal number pairing
- **DoD:** QR code scan completes. `openclaw message send` delivers to WhatsApp.
- **Status:** ✅ Complete
- **Evidence:** All configs: `channels.whatsapp.enabled: true`, `dmPolicy: "open"`, `groupPolicy: "disabled"`. Baileys-based QR pairing. Credentials in `~/.openclaw/credentials/whatsapp/`.

### T1.2.2 — Test inbound: text + voice note
- **DoD:** Text messages appear in gateway log with sender+content. Voice notes appear with .ogg file path. Both <2s.
- **Status:** ✅ Complete
- **Evidence:** `extensions/whatsapp/src/inbound/extract.ts` handles text extraction. `inbound/media.ts` detects `audioMessage` as `audio/ogg; codecs=opus`. Voice notes normalized as `<media:audio>` placeholder.

### T1.2.3 — Test outbound: text reply + audio file reply
- **DoD:** Text reply appears in WhatsApp. Audio plays as voice note (ptt=true). Both <3s.
- **Status:** ✅ Complete
- **Evidence:** `extensions/whatsapp/src/send.ts` rewrites `audio/ogg` → `audio/ogg; codecs=opus` for PTT compatibility. Auto-TTS configured: Edge TTS with `hi-IN-SwaraNeural` voice for Hindi responses. Text chunking at 4000 chars.
