# S2.1: Language Detection & Translation

**Epic:** E2 — Bhashini Language MCP Server
**Priority:** P0 | **Points:** 5 | **Sprint:** W1
**Status:** Complete

## Acceptance Criteria

MCP server exposes detect_language and translate tools. Hindi→English and English→Hindi translation works with <2s latency. Handles empty input, unsupported language, and API timeout gracefully.

## Tasks & Verification

### T2.1.1 — Scaffold bhashini-lang MCP server (FastMCP)
- **DoD:** Empty MCP server starts, responds to `tools/list`. Connects to OpenClaw as registered skill.
- **Status:** ✅ Complete
- **Evidence:** `bhashini_lang/server.py` — FastMCP("bhashini-lang") with 4 `@mcp.tool()` decorated functions. Runs via `python -m bhashini_lang.server`.

### T2.1.2 — Bhashini pipeline helper (Search → Config → Compute)
- **DoD:** Helper takes (task_type, source_lang, target_lang) → pipeline config. Caches with 24hr TTL. Handles API errors with retry then raise.
- **Status:** ✅ Complete
- **Evidence:** `bhashini_lang/client.py:19-74` — `_pipeline_cache` dict with 1-hour TTL (`_PIPELINE_TTL = 3600.0`). Two-step pipeline: auth POST to ULCA → compute POST to callbackUrl. Cache key: `f"{task_type}:{source_lang}:{target_lang}"`.

### T2.1.3 — detect_language tool
- **DoD:** Correctly identifies Hindi, English, Tamil, Telugu. Returns ISO-639 code + confidence. Returns 'unknown' for unsupported.
- **Status:** ✅ Complete
- **Evidence:** `server.py:42-54` — calls provider.detect_language(), returns `LanguageDetection(language, confidence)`. Bhashini backend uses Devanagari character ratio heuristic. Sarvam backend uses `text-lid` API with ISO-639 mapping.

### T2.1.4 — translate tool
- **DoD:** Hi→En and En→Hi: 5 test sentences accurate. Empty→empty. Timeout(>10s)→error.
- **Status:** ✅ Complete
- **Evidence:** `server.py:57-81` — 2-level fallback: primary → secondary provider → passthrough. Bhashini uses ULCA translation pipeline. Sarvam uses `mayura:v1` model with "colloquial" mode.

### T2.1.5 — Unit tests for language tools
- **DoD:** 10+ test cases covering happy path, empty input, unsupported language, API timeout mock, malformed response. All pass.
- **Status:** ✅ Complete (6 tests)
- **Evidence:** `tests/test_server.py` — 4 language detection tests (Hindi, English, mixed, empty/numeric), 1 translation test with mocked pipeline, 1 STT test. Uses `respx` for HTTP mocking and `pytest-asyncio`.
