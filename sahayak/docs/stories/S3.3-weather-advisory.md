## Tasks & Verification

### T3.3.1 — get_weather tool
- **DoD:** Returns weather data for district. LLM generates crop advisory from weather data. Handles missing data gracefully.
- **Status:** ✅ Complete
- **Evidence:**
  - `weather.py:18-71` — accepts `district`, `state`. Returns `WeatherResponse` with `WeatherInfo(district, state, temp_max, temp_min, rainfall, humidity, forecast, advisory)`.
  - IMD data: resource ID `62bc2e75-6840-447e-8835-4f8f6fef2b4c`.
  - Safe float parsing: `safe_float()` handles None, empty, "NA" values without crashing.
  - Missing data: returns `WeatherResponse(message="No weather data found for {district}, {state}")`.
  - API error: returns `WeatherResponse(message="Error fetching weather data: {exc}")`.
  - Retry: uses shared `fetch_with_retry()` (1 retry on 5xx).
  - Cache: 3-hour TTL (`_WEATHER_TTL = 3 * 3600`).
  - **LLM-generated advisory:** The weather data (including raw advisory field from IMD) is returned to the Claude Sonnet agent, which generates contextual farming advice based on the data. This is by design — the system prompt (`SYSTEM.md`) instructs the agent to provide weather-based farming guidance. The advisory is LLM-generated at the agent layer, not hardcoded in the tool.
- **Fix applied:** Added retry-on-5xx via shared `fetch_with_retry`.
