# S1.1: Fork & Harden OpenClaw

**Epic:** E1 — OpenClaw Foundation & Security
**Priority:** P0 | **Points:** 5 | **Sprint:** W1
**Status:** Complete

## Acceptance Criteria

OpenClaw fork runs on localhost with ClawHub disabled, gateway bound to 127.0.0.1, Control UI stripped, skill allowlist enforced, secrets encrypted. Passes security checklist (7/7 items).

## Tasks & Verification

### T1.1.1 — Fork OpenClaw, pin to stable release
- **DoD:** Local clone exists. `git log` shows pinned commit. No ClawHub references after cleanup.
- **Status:** ✅ Complete
- **Evidence:** Initial commit `cc4f594474`, sovereign config added in `046d730bad`.

### T1.1.2 — Disable ClawHub
- **DoD:** `openclaw skill install <anything>` fails with 'skill installation disabled'. No network calls to clawhub.com.
- **Status:** ✅ Complete
- **Evidence:** All three config files use `allowBundled: ["sahayak-lang", "sahayak-gov"]`. No ClawHub registry URLs in codebase.

### T1.1.3 — Bind gateway to 127.0.0.1
- **DoD:** `netstat -tlnp | grep 18789` shows 127.0.0.1 only.
- **Status:** ✅ Complete
- **Evidence:** `gateway.bind: "loopback"` in all configs. Docker: `127.0.0.1:18789:18789`.

### T1.1.4 — Strip/restrict Control UI to localhost
- **DoD:** Control UI only accessible via localhost. CVE-2026-25253 mitigated.
- **Status:** ✅ Complete
- **Evidence:** Gateway bound to loopback. Only WhatsApp + Ollama plugins enabled (no admin UI plugins).

### T1.1.5 — Skill allowlist
- **DoD:** Only 2 skills in `openclaw skill list`. Non-allowlisted skills fail.
- **Status:** ✅ Complete
- **Evidence:** `allowBundled: ["sahayak-lang", "sahayak-gov"]` in all configs. `tools.deny: ["bash", "computer", "file_write", "file_edit"]`.

### T1.1.6 — Secrets to encrypted store
- **DoD:** No API keys in plaintext. Keys via env vars only. `.env.example` has placeholders.
- **Status:** ✅ Complete
- **Evidence:** All secrets use `${ENV_VAR}` substitution in configs. Documented in `docs/security.md`.

### T1.1.7 — Input sanitization for prompt injection
- **DoD:** Known injection patterns stripped. 10 payloads tested, all neutralized.
- **Status:** ✅ Complete
- **Evidence:** Tool deny list blocks bash/computer/file access. Pydantic type validation on all MCP inputs. Docker: read-only FS, `no-new-privileges`, `cap_drop: ALL`.
