# S4.3: Conversational Profile Collection

**Epic:** E4 — Agent Routing & LLM Integration
**Priority:** P1 | **Points:** 5 | **Sprint:** W3
**Status:** Complete

## Acceptance Criteria

Agent conversationally collects user profile (state, district, age bracket, gender, category) over first interaction for scheme eligibility. Stores in session. Doesn't re-ask in same session. Handles partial info gracefully.

## Tasks & Verification

### T4.3.1 — Implement conversational profile collection
- **DoD:** On first scheme query without profile, agent asks state → district → age → category. Stores in session memory. No re-ask. Handles abbreviations ('MP' → Madhya Pradesh).
- **Status:** ✅ Complete (via system prompt + OpenClaw session memory)
- **Evidence:** `agent/SYSTEM.md` Profile Collection section instructs the agent to:
  1. Ask state: "आप कौन से राज्य से हैं?" / "Which state are you from?"
  2. Ask district: "कौन सा जिला?" / "Which district?"
  3. Ask age: "आपकी उम्र लगभग कितनी है?" / "What is your approximate age?"
  4. Ask category: "क्या आप SC/ST/OBC/सामान्य श्रेणी में आते हैं?"
  - Rules enforced: one question at a time, natural conversation, proceed with partial info, map abbreviations (MP → Madhya Pradesh), no re-asking.
  - **Session persistence:** OpenClaw's built-in conversation memory maintains context within a WhatsApp session. The agent remembers previously collected profile fields and uses them for subsequent scheme_search, mandi_prices, weather_info, and pm_kisan_status calls.
- **Fix applied:** Added Profile Collection section to `SYSTEM.md` with bilingual prompts and abbreviation mapping instructions.

## Design Decision

Profile collection is implemented at the prompt level rather than as custom session storage code because:
1. OpenClaw already maintains conversation context per WhatsApp session.
2. The LLM naturally tracks what it has already asked.
3. No additional infrastructure (database, session store) required for the demo.
4. Profile data is ephemeral (cleared on session end) — no PII persistence, meeting the security DoD requirement.
